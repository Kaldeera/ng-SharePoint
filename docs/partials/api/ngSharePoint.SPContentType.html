SPContentType
service in module ngSharePoint

Description

SPContentType factory provides access to all content types (web or list). This factory allows retrieval of associated fields. It also, allows to get and set jsLink properties.

At the moment, not all SharePoint API methods for content type objects are implemented in ngSharePoint

Methods

  • constructor(parentObject, id, contentTypeProperties)

    Instantiates a new SPContentType object for a specific web or list content type in the server. It's possible to specify their properties.

    Parameters
    ParamTypeDetails
    parentObjectobject

    A valid SPWeb or SPList object where the content type is associated.

    idstring

    Content type ID.

    contentTypePropertiesobject

    Properties to initialize the object

    Example

    Use SPList.getContentType and SPList.getContentTypes to retrieve instances of the associated content types.

      list.getContentType('Issue').then(function(issueCt) {

    issueCt.getFields().then(function() {

         angular.forEach(issueCt.Fields, function(field) {
             console.log(field.Title);
         });

    });
  });

  • getFields()

    This method retrieves the Fields collection of the content type and creates a new object property called "Fields" that contains a named property for every field.

    After a call to this method, the schema of every field is available in the content type and all their properties (default values, validation expressions, choice values or lookup properties).

    For a complete list of field properties go to Microsoft SharePoint field api reference. Also, there are additional field specific properties that you can retrieve based on the field type: FieldCalculated, FieldCollection, FieldComputed, FieldDateTime, FieldGeolocation, FieldGuid, FieldLookup and FieldUser, FieldMultiChoice, FieldChoice, and FieldRatingScale, FieldMultiLineText, FieldNumber and FieldCurrency, FieldText, FieldUrl,

    With all of this information, you might construct new interfaces (views, forms, etc) that follow definitions of any SharePoint content type.

    Note: The list of fields of the list isn't necessaray equal to the item content type.

    Returns
    promise

    promise with an object that contains all of the fields schema

    Example

    // a pre-initialized "ct" object ...
ct.getFields().then(function() {

    // at this point, you have access to the definition of any content type field
    console.log(ct.Fields.Title.DefaultValue);
    // this returns '' or any defined value

    console.log(ct.Fields.DueDate.Required);                 
    // this returns true or false

    console.log(ct.Fields.Editor.ReadOnlyField);
    // this returns true

    console.log(ct.Fields.ProjectStatus.Choices.results);
    // this returns an array with available choices ['Open', 'Closed', 'Draft']
});
